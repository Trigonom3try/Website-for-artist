3. The Sprite Editor

The tool that you will be needed to use to create your sprites will be a piece of proprietary software, written in Java, the SpriteEditor_v1.0. You will need to use this tool because the games which will employ the sprites, such as they are, can only read files of the format created by the Editor. The Editor itself should be self-explanatory enough to use, with sliders to create custom colors plus a pre-made pallette of default colors, as well as the ability to use different brushes, customize your brush, and draw lines, circles, and rectangles. Each sprite for this particular project will be 60 pixels x 60 pixels. Sprites are saved into and loaded from text files. Saved files will appear in the same directory as the code for the editor; likewise, in order to load a file, it must be in the same directory. It is important that you do not try to edit these files in any way (I mean, they contain your drawings, so you wouldn't want to anyway). When you finish a sprite, you can email the text file to lemongrab6000@gmail.com.

To run the Editor, you can either:

  1. Click on the link to download the .class file; Compile and run the file from the Windows command line
  2. Click on the link to download the .txt file; rename the file with extension .class; compile and run from the Windows command line
  3. Copy and paste the source code (do not edit unless you know what you're doing!) included below into a .txt file; rename the file with extension .class; compile and run the file from the Windows command line
